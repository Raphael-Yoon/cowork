# ITGC 기준통제별 상세 리뷰 가이드라인 (Full List)

본 문서는 IT 일반통제(ITGC) 실태 점검 및 조서 리뷰를 위한 표준 참조 모델이다. AI(Claude Code, Gemini CLI 등)는 각 통제 코드별 '리뷰 포인트'와 '부적합 기준'을 바탕으로 증적의 적정성을 판단한다.

---

## 1. IT 정책 및 보안 교육 (PWC)

### [PWC01] IT 정책서 관리
- **리뷰 포인트**: IT 운영 프로세스(액세스, 변경, 개발, 운영 등) 정책 포함 여부, 매년 경영정보팀장 검토/승인 여부, 게시판 공지 여부.
- **부적합 기준**: 승인 프로세스 누락, 정책서 내 핵심 통제 영역 누락, 최신화 미비.

### [PWC02] 정보보안 교육
- **리뷰 포인트**: 매분기 전 임직원 대상 교육 실시 증적, 교육 안내 게시물 및 참석자 명단.
- **부적합 기준**: 분기별 주기 미준수, 교육 대상자 누락, 교육 자료 미비.

---

## 2. 계정 및 권한 관리 (APD)

### [APD01] Application 권한 승인
- **리뷰 포인트**: 신규/변경 요청 시 부서장/IT팀장 사전 승인. 모집단과 승인 문서 대조.
- **부적합 기준**: 사후 승인, 승인권자 적정성 미달(대결 등 절차 미비).

### [APD02] Application 부서이동자 권한 회수
- **리뷰 포인트**: 인사 발령 후 즉시 회수 여부. 인사 명단과 권한 삭제 로그 대조.
- **부적합 기준**: 전보 후 기존 권한 유지, 회수 처리 지연.

### [APD03] Application 퇴사자 접근권한 회수
- **리뷰 포인트**: 퇴사 시 계정 자동 삭제/비활성화 로직 또는 수동 처리 적시성.
- **부적합 기준**: 퇴사자 계정 활성화 상태 유지, 퇴사 후 로그인 이력 존재.

### [APD04] Application 권한 Monitoring
- **리뷰 포인트**: 전체 사용자 권한 정기 점검 이력, 책임자 승인.
- **부적합 기준**: 모니터링 주기 미준수, 부적정 권한 발견 시 조치 미이행.

### [APD05] Application 관리자 권한 제한
- **리뷰 포인트**: 관리자 권한 부여 인원의 최소화 및 적정성.
- **부적합 기준**: 비관련 부서 인원의 관리자 권한 보유.

### [APD06] Application 패스워드 (자동)
- **리뷰 포인트**: 최소 8자리 이상, 문자/숫자/특수문자 조합 설정 화면 확인.
- **부적합 기준**: 시스템 설정값이 정책보다 완화되어 있음.

### [APD07] Data 변경 승인
- **리뷰 포인트**: DB 직접 수정 시 사전 승인 및 결과 확인.
- **부적합 기준**: 승인 없는 DB 변경(DML), 사유 불분명.

### [APD08] Data 변경 권한 제한
- **리뷰 포인트**: 데이터 변경 가능 인원의 최소화.
- **부적합 기준**: 일반 사용자에게 DB 수정 권한 부여.

### [APD09] OS 접근권한 승인
- **리뷰 포인트**: OS 계정 생성/변경 시 적절한 승인권자 승인.
- **부적합 기준**: 승인 증적 없는 OS 계정 생성.

### [APD10] OS 패스워드 (자동)
- **리뷰 포인트**: 회사 규정에 따른 최소자리, 복잡성 설정값 확인.
- **부적합 기준**: 복잡성 설정 미적용.

### [APD11] OS 관리자 권한 제한
- **리뷰 포인트**: OS Super User(Root 등) 권한자 최소화.
- **부적합 기준**: 불필요한 인원의 Root 권한 보유.

### [APD12] DB 접근권한 승인
- **리뷰 포인트**: DB 접근 권한 부여 시 사전 승인.
- **부적합 기준**: 승인 절차 생략.

### [APD13] DB 패스워드 (자동)
- **리뷰 포인트**: DB 계정 패스워드 정책 설정값 확인.
- **부적합 기준**: 정책 미준수.

### [APD14] DB 관리자 권한 제한
- **리뷰 포인트**: DB Super User 권한자 최소화.
- **부적합 기준**: DBA 외 인원의 Super User 권한 보유.

## 3. 프로그램 변경 관리 (PC)

### [PC01] 프로그램 변경 승인
- **리뷰 포인트**: 변경요청서 작성, 요청부서장/IT팀장 승인.
- **부적합 기준**: 공식 요청서 없는 변경, 사전 승인 누락.

### [PC02] 프로그램 변경 사용자 테스트
- **리뷰 포인트**: 현업 사용자 인수 테스트(UAT) 수행 및 결과 문서화.
- **부적합 기준**: 테스트 미실시, 테스트 결과 부적합함에도 배포.

### [PC03] 프로그램 변경 이관 승인
- **리뷰 포인트**: 운영 환경 이관 전 최종 승인권자 승인.
- **부적합 기준**: 이관 후 승인(사후 승인).

### [PC04] 개발/운영 환경의 분리 (자동)
- **리뷰 포인트**: 개발과 운영 환경의 논리적 분리 상태 확인.
- **부적합 기준**: 개발자가 운영 서버에 직접 접속/수정 가능.

### [PC05] 이관담당자 권한 제한
- **리뷰 포인트**: 개발자와 이관담당자 직무 분리(SoD).
- **부적합 기준**: 동일인이 개발하고 운영에 직접 배포.

### [PC06] OS 설정변경
- **리뷰 포인트**: OS 설정 변경 시 승인 절차 준수.
- **부적합 기준**: 임의의 OS 파라미터 변경.

### [PC07] DB 설정변경
- **리뷰 포인트**: DB 설정 변경 시 승인 절차 준수.
- **부적합 기준**: 승인 없는 DB 환경 설정 변경.

## 4. IT 운영 관리 (CO)

### [CO01] Batch Schedule 등록/변경 요청 및 승인
- **리뷰 포인트**: 배치 등록/변경 시 요청서 및 승인 증적.
- **부적합 기준**: 임의의 배치 등록.

### [CO02] Batch Schedule 등록/변경 권한 제한
- **리뷰 포인트**: 권한이 지정된 담당자로 제한되어 있는지 확인.
- **부적합 기준**: 비담당자의 배치 수정 권한 보유.

### [CO03] Batch Schedule 모니터링
- **리뷰 포인트**: 오류 내역 인지 및 조치 결과 문서화.
- **부적합 기준**: 배치 오류 방치, 조치 이력 부재.

### [CO04] 백업 모니터링
- **리뷰 포인트**: 주기적 백업 수행 로그, 백업 미디어 소산 보관 증적.
- **부적합 기준**: 백업 실패 미조치, 소산 기록 불일치.

### [CO05] 장애 관리
- **리뷰 포인트**: 복구 절차 정기 테스트 실시 및 결과 보고서.
- **부적합 기준**: 복구 테스트 미실시.

### [CO06] 데이터센터 접근 제한
- **리뷰 포인트**: 외부인 출입 명부 작성, 담당자 동행 여부.
- **부적합 기준**: 무단 출입 또는 기록 누락.

---

## 5. 프로젝트 관리 (PD)

### [PD01] 사용자 인수 테스트
- **리뷰 포인트**: 프로젝트 단계별 테스트(단위/통합/UAT) 결과보고서 승인.
- **부적합 기준**: 테스트 증적 미비, 결함 미조치 상태의 완료 보고.

### [PD02] 데이터 이관
- **리뷰 포인트**: 이관 데이터 정합성 검증 테스트 결과 문서화.
- **부적합 기준**: 이관 전/후 데이터 불일치.

### [PD03] 이슈 관리
- **리뷰 포인트**: 프로젝트 기간 내 이슈/오류 관리 대장 현행화.
- **부적합 기준**: 미해결 이슈의 방치.

### [PD04] 사용자 교육
- **리뷰 포인트**: 매뉴얼 배포 및 사용자 교육 실시 증적.
- **부적합 기준**: 교육 증적 부재.

---

## 6. 지원 도구 관리 (ST)

### [ST01] Supporting Tool Super User 권한 제한
- **리뷰 포인트**: 관리 도구(DBSafer 등)의 슈퍼유저 인원 제한.
- **부적합 기준**: 불필요한 관리자 계정 남발.

### [ST02] Supporting Tool 패스워드 설정
- **리뷰 포인트**: 복잡성, 최소길이, 변경주기(분기) 설정 확인.
- **부적합 기준**: 정책 미설정 또는 기본 패스워드 사용.

### [ST03] Supporting Tool 기능 변경 요청 및 승인
- **리뷰 포인트**: 도구 내 프로그램/설정 변경 시 승인 및 테스트.
- **부적합 기준**: 임의의 보안 설정 변경.

## 🛠 AI Auditor 공통 검토 원칙
1. **수동 통제**: 반드시 모집단의 완전성(IPE) 확인 후 표본(Sampling) 검사 수행.
2. **자동 통제**: 설정값 스크린샷 1건으로 검증하되, 기간 내 유효성 확인.
3. **직무 분리**: 모든 승인 프로세스에서 요청자와 승인자가 동일인인지 체크.
